# Livrables

Tous les documents et contenus produits dans les sessions Jarvis. Chaque dossier correspond à un projet réel de Raphaël.

## Structure

```
livrables/
│
├── business-ia/                    → Projet prioritaire : lancer l'agence IA avant le 16 sept.
│   ├── etude-de-marche/            → Analyse du marché, concurrents, niches identifiées
│   ├── offre-et-positionnement/    → L'offre construite, le positionnement, les prix
│   ├── business-plan/              → Business plan complet + montage financier
│   ├── prospection-clients/        → Listes prospects, scripts d'approche, suivi
│   └── scripts-et-automations/     → Outils IA et automatisations créés pour les clients
│
├── immobilier/                     → Projets immo
│   ├── mondeville/                 → Appart en cours : achat (10 juil.), travaux, mise en loc (mi-août)
│   └── projets-futurs/             → Recherches et préparation des prochains biens
│
├── alternance-btp/                 → Rosay Techniques Couvertures
│   ├── documents-chantier/         → PPSPS, plannings, extractions de baux, comptes rendus
│   ├── rapports-et-comptes-rendus/ → Rapports d'avancement, réunions de chantier
│   └── formation-ecole/            → BTP Builders : cours, partiels, mémoire
│
├── organisation/                   → Système d'organisation personnel
│   ├── planning-ete/               → Planning détaillé de l'été 2026
│   └── routines/                   → Suivi quotidien : sport, anglais, lecture, IA
│
└── formation-ia/                   → Monter en compétence IA
    ├── notes-de-cours/             → Notes de la formation 6h + autres ressources
    └── ressources/                 → Articles, outils, références utiles
```

## Convention de nommage

`AAAA-MM-JJ_nom-du-fichier.ext`

Exemple : `2026-07-01_offre-agence-ia-btp.md`

## Pour Claude

Quand tu produis un livrable, propose systématiquement de l'enregistrer dans le bon sous-dossier ci-dessus. Si le contexte ne rentre dans aucune catégorie, crée un sous-dossier adapté et mets à jour ce README.
